Spell System

Inspired by Zelda
Tears of the Kingdom

About

In order to develop more advanced and complexe interaction between the player and its environement, we created this mini-project trying to reproduce (in a simplified way) some of the Zelda ToTK Spells. This allowed us to work on the following aspects:

  • Reverse engineering 4 skills from the game : Ascend, Fuse, Recall and Ultrahand;
  • Building a modulable and evolutive code baseline, by coding a Spell class foundation using Unity's Scriptable Objects and adding a global Spell Controller;
  • Adding a basic UI for spell selection;
  • Developing and adding visual effects to reinforce player's immersion, using Unity's Graph System for Particles, and explored shaders with the Graph System for Shaders, to create electric arcs, trails and outlines effects;
  • Finally, building an entire scene to highlight each of those developments.

To learn more about the implementations choices you may refer to the project's Github.

Ascend

This ability takes the player to a portal leading through surfaces. Various constraints have been implemented for this spell such as basic player movement animation through the portals, raycast checks, and visual effects.

Fuse

Allow the player to aggregate other entities to its own weapon and create new behaviour and interactions with the environment.

In this exercise, a torch tagged with Fire element is added to our weapon to activate the mechanism to reach the next stage.

Recall

Recallable objects record their position at a certain frame rate. The recall spell stops the time and then allows the player to retrospecte objects position.

The spell activation allows the player to rewind the objects position with different ease-types between those positions.

Ultrahand

Finally, the ultrahand spell helps the player to create new interactions and structures physics by linking objects together. In this exemple we build a platform allowing the player to use a zipline. The Ultrahand spell is by far the most complexe spell to apprehend. Many improvements can be further realized, such as being able to separate objects previously linked by the spell, a better control of the camera, different directions to join objects together depending on objects faces normals rather than the point of contact, and so on.

Open World
Mini-Project

About

A Third Person Open World mini-project introducing me to all different kind of problematics to develop and solve ! Some of the tasks realized in this project :

  • Implement the UI for the different menus and the inventory;
  • Architecture the State Machine for the Enemies, using Scriptable Objects allowing for code flexibility and maintainability with respect to the SOLID principles;
  • Program all the Player Controllers to describe its behaviour and all interaction with its environnement (player inputs, entities, objects, events, ...);
  • Dive into the Unity's Animator to control our player and enemies animation depending on their states and actions;
  • Add PNJ and basic quests and dialogs systems based on Unity's Scriptable Objects for modularity and generalized behaviours;
  • Prepare simple cutscenes to introduce new zones or objectives for the player to reach;
  • Manipulate Unity's optimizations solutions such as partial scene loading and occlusion culling;
  • And many more, from adding visual and sound effects, adapt level design, make a save system, allow for player customization...

Source code is available on Github.

Tower Defense
Mini-Project

About

Here we developed a very classic tower defense game, to work on wave management, event systems, and overall a better OOP architecture.

  • Simple UI systems to build, upgrade, and sell the turrets;
  • A gold manager that interacts with the player actions and the mob deaths;
  • Basic AI path finding system for the mob to reach the Tower;
  • Waves system, adjustable mob spawning characteristics;
  • Two types of turrets : projectiles for burst damage and flamethrower for dps damages;
  • Strong emphasis on design patterns implementation (Factories, Builder, Observers, Object Pooling, ...);
  • Develop our own level builder to design and create our levels faster !

Source code is available on Github.

Survival Horror
Mini-Project

About

Heavily inspired by my souvenirs of the Zombie mode from Call of Duty - Black Ops and more specifically by the map Kino Der Toten, but in a 2D fashion !

This mini-project helped me learning and improving my 2D programming skills using Gms2 :

  • Designing a 2D map and tile collisions based behaviours;
  • Basic enemy AI chasing behaviour using GameMaker path finding system;
  • Implementing a complete State Machine for the player;
  • Coding a generic gun object for diverse firing behaviours, allowing to control many properties such as knockback, firing cooldown, projectiles number, dispersion, range and speed, automatic shooting, and so on;
  • A simplistic wave system manager, relying on unlocked enemy spawns and a glimpse of randomness to make each run unique !
  • Adding plenty of cool sound and visual effects from talented artists !

Source code is available on Github.
